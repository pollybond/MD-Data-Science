# Требования и реализация сервиса

## 1. Реализация сервисов

Должно быть реализовано как минимум два сервиса:

- **Сервис управления пользователями**: Операции создания, поиска и управления пользователями.
- **Бизнес-сервис**: Операции с маршрутами и поездками, включая создание маршрутов, поездок, подключение пользователей к поездкам и получение информации о поездках.

## 2. Аутентификация с использованием JWT-token

Аутентификация с использованием JWT-token реализована следующим образом:

- **Маршрут для получения токена**: `/token`
- **Функция `get_current_client`**: Проверяет JWT-токен и возвращает имя пользователя, если токен действителен.
- **Функция `create_access_token`**: Создает JWT-токен на основе данных пользователя.

## 3. Поддержка GET/POST методов

В коде реализованы как GET, так и POST методы:

- **POST**:
  - Создание пользователя
  - Создание маршрута
  - Создание поездки
  - Получение токена

- **GET**:
  - Поиск пользователя по логину
  - Поиск пользователя по маске имени и фамилии
  - Получение маршрутов пользователя
  - Получение информации о поездке

## 4. Хранение данных в памяти

Данные хранятся в памяти в виде списков:

- **`users_db`**: Список пользователей.
- **`routes_db`**: Список маршрутов.
- **`trips_db`**: Список поездок.

## 5. Мастер-пользователь

В целях проверки должен быть заведён мастер-пользователь с именем `admin` и паролем `secret`.

Мастер-пользователь заведен в псевдо-базе данных `client_db`:
